# Google Play Console サブスクリプション設定手順

Morincumアプリ（Android）でのサブスクリプション課金を有効にするための手順書です。  
この作業は **EASビルド・ストア申請の前** に完了させてください。

---

## 前提条件

- [ ] **Google Play Console** アカウント作成済み（初回登録費 $25）
- [ ] Google アカウントでログイン済み
- [ ] [Google Play Console](https://play.google.com/console) にアクセス可能
- [ ] [Google Cloud Console](https://console.cloud.google.com) にアクセス可能

## 1. アプリ登録

### 1-1. 新規アプリ作成

1. [Google Play Console](https://play.google.com/console) → 「アプリを作成」
2. 設定:

| 項目 | 値 |
|---|---|
| アプリ名 | `配当の森 by MORINCUM` |
| デフォルトの言語 | 日本語（ja）|
| アプリまたはゲーム | アプリ |
| 無料または有料 | 無料 |

3. 宣言事項に同意 → 「アプリを作成」

> **パッケージ名**: `com.morincum.haitonomori`  
> ※ EASビルド時に `app.json` の `android.package` として設定されます。

### 1-2. アプリ情報の入力

「ストアの設定」→「アプリの詳細情報」:

| 項目 | 値 |
|---|---|
| カテゴリ | ファイナンス |
| メール | サポート連絡先メールアドレス |

「ストアの設定」→「コンテンツのレーティング」:
- アンケートに回答してレーティングを取得（暴力・成人コンテンツなし → 全年齢対象）

「ストアの設定」→「プライバシーポリシー」:
- URL を入力（例: `https://morincum.jp/privacy`）

## 2. サブスクリプション商品登録

> ⚠️ **重要**: Google Play のサブスクリプションを作成するには、  
> **少なくとも1回 APK/AAB をアップロード済み** である必要があります。  
> EASビルドで Internal Testing トラックにアップロードした後に実施してください。

### 2-1. サブスクリプション作成

1. 「収益化」→「定期購入」→「定期購入を作成」
2. 設定:

| 項目 | 値 |
|---|---|
| 製品ID | `standard_monthly` |
| 名前 | `Standardプラン` |
| 説明 | `広告非表示・CSVエクスポート・バックアップ機能が使えるプレミアムプランです。` |

3. 「ベースプランとオファーを追加」

### 2-2. ベースプラン設定

| 項目 | 値 |
|---|---|
| ベースプランID | `monthly` |
| 自動更新 | 自動更新 |
| 請求期間 | 毎月 |
| デフォルト価格 | ¥480 |

> 💡 日本価格を入力すると他国の価格が自動換算されます。必要に応じて調整してください。

### 2-3. 無料トライアル設定（オプション）

ベースプランの「オファーを追加」:

| 項目 | 値 |
|---|---|
| オファーID | `free-trial` |
| オファータイプ | 無料トライアル |
| 期間 | 7日間 |
| 適用対象 | 新規ユーザー |

### 2-4. 有効化

設定完了後「有効化」→ ステータスが「有効」になることを確認。

## 3. サービスアカウント作成

RevenueCat が Google Play のレシートを検証するために必要です。

### 3-1. Google Cloud でサービスアカウントを作成

1. [Google Cloud Console](https://console.cloud.google.com) にアクセス
2. Google Play Console と **同じ Google アカウント** で作成したプロジェクトを選択  
   （なければ新規プロジェクトを作成）
3. 「IAM と管理」→「サービスアカウント」→「サービスアカウントを作成」
4. 設定:

| 項目 | 値 |
|---|---|
| サービスアカウント名 | `revenuecat` |
| サービスアカウントID | `revenuecat` |
| 説明 | `RevenueCat billing integration` |

5. ロールを付与:
   - 「ロールを選択」→ **「Pub/Sub サブスクライバー」** を追加  
     （Google Play Developer API 用）
6. 「完了」

### 3-2. JSONキーを生成

1. 作成したサービスアカウント → 「キー」タブ → 「鍵を追加」→「新しい鍵を作成」
2. キーのタイプ: **JSON**
3. ダウンロードされた `.json` ファイルを保管（**再ダウンロード不可**）

### 3-3. Google Play Console でサービスアカウントを連携

1. Google Play Console → 「設定」→「API アクセス」
2. 「Google Cloud プロジェクトをリンク」（上記のプロジェクトをリンク）
3. サービスアカウント一覧に先ほど作成したアカウントが表示される
4. 「アクセスを許可」→ 権限を設定:

| 権限 | 設定 |
|---|---|
| アプリ情報の表示（読み取り専用） | ✅ |
| 財務データの表示 | ✅ |
| 注文と定期購入の管理 | ✅ |

5. 「ユーザーを招待」→「アクセス許可を適用」

> これらの `.json` キーは RevenueCat の設定（STEP 7-2）で使用します。

## 4. ライセンステスト設定

本番課金なしでサブスクリプション購入をテストするために必要です。

### 4-1. ライセンステスターの追加

1. Google Play Console → 「設定」→「ライセンステスト」
2. テスト用 Gmail アドレスを追加

> ライセンステスターは **本番課金なし** でサブスクリプションを購入できます。  
> テスト端末に該当の Gmail アカウントを登録して使用します。

### 4-2. Internal Testing トラックの設定

1. 「リリース」→「テスト」→「内部テスト」→「テスターを管理」
2. テスターのメールアドレスリスト（Google グループまたは直接入力）を追加
3. テスターに共有された URL からアプリをインストールするよう案内

> ⚠️ Internal Testing のサブスクリプションは **課金されません**。  
> テスト完了後は Open / Closed Testing に移行してください。

## 5. アプリ申請準備

### 5-1. ストア掲載情報

「メインのストア掲載情報」:

**短い説明（80文字以内）:**
```
株式・ETFの配当金を管理するポートフォリオアプリ
```

**詳しい説明（4000文字以内）:**
```
配当の森は、株式・ETFの配当金を管理する家計簿アプリです。

【主な機能】
・配当金の年間・月別集計
・銘柄別ポートフォリオ管理
・NISA口座管理
・複数口座対応（無料: 1口座、Standard: 3口座）
・為替レート対応（米国株・ETF）
・CSVエクスポート（Standardプラン）

【Standardプラン（月額¥480）】
・広告非表示
・3口座まで管理可能
・CSVエクスポート
・データバックアップ
```

### 5-2. グラフィック素材

| 素材 | サイズ |
|---|---|
| アイコン | 512 × 512px（PNG、アルファなし） |
| フィーチャーグラフィック | 1024 × 500px |
| スクリーンショット（電話） | 最低2枚（縦向き推奨） |
| スクリーンショット（タブレット） | 任意 |

### 5-3. データセーフティの入力

「ストアの設定」→「データセーフティ」:

| 質問 | 回答 |
|---|---|
| ユーザーのデータを収集・共有しますか？ | はい |
| データの種類: 個人情報（メールアドレス） | 収集（アカウント管理のため） |
| データの暗号化転送 | はい（HTTPS） |
| データ削除リクエストの提供 | はい（アカウント削除機能あり） |
