## 1. ブランドコンセプト

> 「どんぐりを集めるように、配当金を少しずつ積み上げていく」

森の中でどんぐりを集めるリス・モーリスをモチーフに、**自然・成長・安心感**を表現する。
派手さより**落ち着いた緑**を基調とし、長期投資という地道な行動を後押しするトーンを大切にする。

### 2-1. テーマカラー（メイン）

| トークン名 | HEX | 用途 |
|-----------|-----|------|
| `accent` | `#5BAD60` | メインアクション・アクティブ状態・グラフ・**現状値の数値表示** |
| `accentSoft` | `#5BAD6020` | アクセントの背景・タグ・ハイライト |
| `gold` | `#F5A623` | **目標・達成率・目標関連数値の表示**・強調表示・達成バッジ |
| `goldSoft` | `#F5A62320` | ゴールドの背景 |
| `nisa1` | `#4A90D9` | NISA成長投資枠 |
| `nisa2` | `#9B7FD4` | NISAつみたて投資枠・優待（株主優待） |
| `usd` | `#5BAD60` | 米国株（USD表示） |
| `red` | `#E05C7A` | エラー・警告・削除 |

### 2-1-a. 現状 vs 目標のカラールール

年間配当金カードなど数値を現状と目標で比較する箇所では、次のカラールールに従う。

| 種別 | トークン | HEX | 例 |
|------|---------|-----|----|
| **現状（実績）** | `accent` | `#5BAD60` | 現在の年間配当、現状月収、現状バーの塗り |
| **目標・達成率関連** | `gold` | `#F5A623` | 目標達成率%、株数目標達成時金額、「あと ¥X」、「目標まで +¥X」 |

- **現状トグル（現状タブがアクティブ）** → `accent`（緑）背景
- **目標トグル（目標タブがアクティブ）** → `gold`（オレンジ）背景

## 3. タイポグラフィ

### 3-1. フォント

```
'Noto Sans JP', 'Hiragino Sans', sans-serif
```

日本語表示を優先。React Nativeでは `NotoSansJP` または `HiraginoSans` を指定。

### 3-2. フォントサイズスケール

| 用途 | サイズ | fontWeight |
|------|--------|------------|
| 特大数値（配当金額等） | 28〜32px | 800 |
| 大見出し | 20〜24px | 800 |
| 中見出し・タイトル | 15〜16px | 700〜800 |
| 本文・ボタン | 13〜14px | 400〜700 |
| サブテキスト・ラベル | 11〜12px | 400 |
| キャプション・バッジ | 8〜10px | 400〜700 |

### 3-3. letterSpacing

| 用途 | 値 |
|------|---|
| アプリ名・ラベル | `1` |
| 大きな数値 | `-0.3`（詰め気味） |

## 4. スペーシング

### 4-1. 基本単位

`4px` を基本単位とし、4の倍数で統一する。

| トークン | 値 | 用途 |
|---------|-----|------|
| xs | 4px | アイコンと文字の間隔 |
| sm | 8px | 関連要素間 |
| md | 12〜16px | カード内パディング |
| lg | 20px | セクション間・カード外パディング |
| xl | 24〜32px | モーダル・画面パディング |

### 4-2. 画面パディング

```
画面横パディング: 16px
コンテンツ下部余白: 120px（フッター分）
カード内パディング: 20px
```

### 4-3. 要素間ギャップ

```
カード内リスト: gap 8〜10px
セクション間: gap 12〜16px
ボタン間: gap 8〜10px
```

## 6. コンポーネント

### 6-1. カード

```
background: card (#FFFFFF)
borderRadius: 20px
padding: 20px
border: 1px solid border (#D8EAD8)
```

### 6-2. プライマリボタン

**Active（通常状態）**

```
background: linear-gradient(135deg, #5BAD60, #388E3C)
color: bg (#F4F7F4)
borderRadius: 12px
padding: 12〜14px 0
fontWeight: 800
fontSize: 13〜14px
```

**Disabled（押せない状態）**

```
background: border (#D8EAD8)
color: #AABFAA
borderRadius: 12px
padding: 12〜14px 0
fontWeight: 800
fontSize: 13〜14px
```

- 背景をテーマの `border` 色（`#D8EAD8`）に、文字をくすんだ緑（`#AABFAA`）にする
- グラデーションは適用しない（単色）
- 使用場面: 必須入力が未入力の場合、処理中の二重送信防止など

### 6-3. セカンダリボタン

```
background: card (#FFFFFF)
color: muted (#7A9A7A)
border: 1px solid border (#D8EAD8)
borderRadius: 12px
padding: 11px 0
fontWeight: 700
```

### 6-4. アクセントボタン（ゴースト）

```
background: accentSoft (#5BAD6020)
color: accent (#5BAD60)
border: 1px solid accent44
borderRadius: 12px
```

### 6-5. 危険ボタン

```
background: #E05C7A11
color: red (#E05C7A)
border: 1px solid red44
borderRadius: 12px
```

### 6-6. FABボタン

```
size: 64 × 64px
borderRadius: 50%
background: linear-gradient(135deg, #5BAD60, #00A882)
boxShadow: 0 4px 24px accent88
position: fixed bottom-right
zIndex: 20
```

### 6-7. 入力フィールド

```
background: bg (#F4F7F4)
border: 1px solid accent55（フォーカス時）または border（通常）
borderRadius: 10px
padding: 10px 12px
color: text (#2D4A2D)
fontSize: 16px
```

### 6-8. トグルスイッチ

```
width: 44px / height: 26px
borderRadius: 13px
ON: background accent (#5BAD60)
OFF: background border (#D8EAD8)
つまみ: 20×20px 白丸、top:3px, left:3px(OFF)→21px(ON)
transition: left 0.2s
```

### 6-9. バッジ・タグ

```
fontSize: 10〜11px
padding: 2px 8px
borderRadius: 6px
背景: カテゴリ色の20%透過
テキスト: カテゴリ色
```

## 11. アクセシビリティ

- タップターゲットは最小 **44×44px** を確保
- テキストと背景のコントラスト比 **4.5:1以上** を目標
- フォントサイズは最小 **10px**（キャプション）、本文は **13px以上**

## 13. エラー・空状態のデザイン

### 13-1. 空状態（Empty State）

| 画面 | メッセージ | CTA |
|------|-----------|-----|
| ホーム（銘柄0件） | まだ銘柄がありません ＋ボタンから追加しよう！ | 銘柄を追加する |
| 計画画面 | 銘柄を追加するとシミュレーションが始まります | 銘柄を追加する |
| 検索結果なし | 見つかりませんでした | 条件を変えて探す |

### 13-2. エラー状態

| 種類 | 表示方法 | カラー |
|------|---------|--------|
| 通信エラー | 全画面エラー＋再試行ボタン | `red (#E05C7A)` |
| 部分的なエラー | カード内にインラインメッセージ | `red (#E05C7A)` |
| NISA枠超過警告 | ⚠️ アイコン＋黄色テキスト | `gold (#F5A623)` |
| 削除確認 | 確認モーダル（取り消し不可の操作） | `red (#E05C7A)` |

## 14. バリデーションの表示ルール

### 14-1. タイミング

| タイミング | 内容 |
|-----------|------|
| 送信時（onSubmit） | 全項目を一括チェック。エラーがある項目にフォーカス |
| フォーカスアウト時 | その項目のみチェック（数値範囲・必須など） |
| 入力中 | 文字数カウントのみ表示（リアルタイムエラーは出さない） |

### 14-2. エラー表示スタイル

- 入力フィールドの `border` を `red (#E05C7A)` に変更
- フィールド直下に赤テキストでメッセージを表示（`fontSize 11px`）

| 項目 | エラー文言 |
|------|-----------|
| 必須入力 | ⚠️ 銘柄名を入力してください |
| 数値範囲外 | ⚠️ 0以上の数値を入力してください |
| NISA枠超過 | ⚠️ NISA枠の上限を超えています |
| PIN不一致 | ⚠️ 一致しません。もう一度入力してください |
| PIN誤入力 | ⚠️ PINが違います |

## 15. ローディング表示

| 種類 | 使用場面 | 表示方法 |
|------|---------|---------|
| 全画面ローディング | 初回データ取得・アプリ起動時 | 中央にスピナー＋「読み込み中...」 |
| スケルトン | カード・リストの読み込み | グレーの矩形でコンテンツ形を模倣 |
| インラインスピナー | ボタン押下後の処理中 | ボタン内に小さいスピナー表示 |
| プルリフレッシュ | リスト更新 | 上引っ張りで更新（ネイティブUX） |

- スピナーカラー: `accent (#5BAD60)`
- スケルトンカラー: `border (#D8EAD8)` → `accentSoft` のグラデーションでシマー

## 20. NISA カラールール

| 枠 | トークン | HEX | 用途 |
|----|---------|-----|------|
| NISA 成長投資枠 | `nisa1` | `#4A90D9` | 成長枠のバー・バッジ・テキスト |
| NISA つみたて投資枠 | `nisa2` | `#9B7FD4` | つみたて枠のバー・バッジ・テキスト |

| 状態 | 表示方法 |
|------|---------|
| 残枠あり | 通常表示（`nisa1` / `nisa2` カラー） |
| 残枠わずか（10%以下） | `gold (#F5A623)` に切替 + ⚠️ アイコン |
| 枠超過 | `red (#E05C7A)` に切替 + エラーメッセージ |
